Test Sistema Autocompilazione Token

Scenario 1: Token da URL (primo accesso QR)

Simula la scansione di un QR code con token nell'URL

Test Login con Token Marco nell'URL
Risultato atteso:
  • Banner verde "Token rilevato automaticamente"
  • Campo username precompilato con il token
  • Campo evidenziato in verde
  • Placeholder "Token precompilato automaticamente"
  • Label "Nome utente (Token)"

Scenario 2: Token da localStorage (PWA persistente)

Simula un accesso PWA con token salvato localmente



2. Test Login con Token da localStorage
Risultato atteso:
  • Stessi indicatori visuali del Scenario 1
  • Token caricato automaticamente dal localStorage

Scenario 3: Login normale senza token

Test Login Normale (senza token)
Risultato atteso:
  • Nessun banner di autocompilazione
  • Campo username vuoto
  • Placeholder "Inserisci il tuo nome utente"
  • Label "Nome utente" (senza "(Token)")

Scenario 4: Sessione scaduta

Test Sessione Scaduta
Risultato atteso:
  • Banner rosso "Sessione scaduta"
  • Token localStorage pulito
  • Nessuna autocompilazione
